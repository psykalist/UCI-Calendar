# Men's UCI Road Calendar

A progressive web app (PWA) showing the live men's UCI road race calendar with stage results, classifications, startlists, and a fantasy league. Hosted on GitHub Pages — no backend required.

**Live app:** https://psykalist.github.io/UCI-Calendar/

---

## Quick start

```bash
# 1. Install Python dependencies (one-time)
pip install requests beautifulsoup4

# 2. Run the scraper to update race data
py scraper.py

# 3. Commit and push to deploy
git add data.json
git commit -m "chore: update race data $(date -u +%Y-%m-%d)"
git push
```

---

## Scripts reference

### `scraper.py` — main data scraper (run daily)
Fetches the full UCI race calendar from CyclingFlash, scrapes stage results, classifications, startlists, rider profiles, and team rosters. Writes `data.json`.

```bash
py scraper.py
```

**What it does (in order):**
1. Discovers all UCI races from CyclingFlash calendar (~10 s)
2. Fetches race info, stage results, GC/Points/Mountain/Youth classifications (3–8 min)
3. Scrapes 35 WorldTeam + ProTeam rosters (~800 riders, ~2 min)
4. Fetches up to 50 new rider profiles incrementally (~1 min)
5. Fetches startlists for races within 21 days (~30 s)
6. Writes `data.json`

Run after major race stages finish to keep data current.

---

### `scrape_teams.py` — team roster scraper (run once per season)
Scrapes all WorldTeam and ProTeam rosters. Only needs re-running if rosters change mid-season. Results are merged into `data.json`.

```bash
py scrape_teams.py
```

---

### `check_and_fix.py` — data integrity checker
Validates `data.json` for common issues (missing fields, broken stage links, etc.) and auto-fixes what it can.

```bash
py check_and_fix.py
```

---

### `detect_changes.py` — change detector
Compares the current `data.json` against the last committed version and prints a summary of what changed (new results, new races, etc.). Useful before pushing.

```bash
py detect_changes.py
```

---

### `pre_push_check.py` — pre-push validator
Runs a set of checks before pushing: validates JSON structure, checks for required fields, warns about anomalies.

```bash
py pre_push_check.py
```

---

### `push.bat` / `git-push.sh` — deploy helpers
Convenience scripts that run the scraper, commit `data.json`, and push to GitHub in one step.

**Windows:**
```bat
push.bat
```

**Git Bash / Linux:**
```bash
bash git-push.sh
```

---

### `check.bat` / `check.sh` — quick health check
Runs `pre_push_check.py` and prints a summary. Use before manual pushes.

---

### `update.bat` — full update cycle (Windows)
Runs scraper → check → commit → push in sequence.

## Deployment

The app is a static site hosted on **GitHub Pages** from the `main` branch root.

```bash
git add data.json index.html sw.js
git commit -m "feat/fix/chore: description (vXX)"
git push
```

GitHub Pages deploys within ~1 minute of the push. Check status at:
`https://github.com/psykalist/UCI-Calendar/actions`

## Push notifications (v22+)

The bell button in the app header enables browser push notifications. When enabled:
1. The browser prompts for notification permission
2. A `push_subscriptions.json` file is downloaded — move it to the project folder
3. The scraper reads this file and sends push notifications when new stage results are detected

VAPID keys are stored in `vapid_keys.json` (excluded from git — keep this file safe).

## Fantasy league

- Pick 9 riders per race before it starts
- Points are scored from stage results (1st = 10pts, 2nd = 7pts, etc.)
- Teams are race-keyed — each race has a separate team
- Export your team code to share with others; import codes to compare teams
- All data is stored in browser `localStorage` (device-local, no backend)

## Known limitations

See [KNOWN_GAPS.md](KNOWN_GAPS.md) for a full list. Key points:
- No real-time live race tracking (results appear after stage finishes)
- Fantasy teams are device-local only (no cloud sync)
- PCS occasionally rate-limits the scraper — just re-run if a fetch fails
